Build From Source - Linux

建立工具和库

  1. 打开一个终端窗口并且执行以下命令：

sh $ sudo apt-get install build-essential libgtk2.0-dev libssl-dev xsltproc ia32-libs libxml2-dev libcap-dev 2. 如果要在一个 64 为操作系统上建立一个 32 位 AllJoyn™，需要安装这些开发库： sh $ sudo apt-get install gcc-multilib g++-multilib libc6-i386 libc6-dev-i386 libssl-dev:i386 libxml2-dev:i386

注意: 如果安装了 32 位的 libssl-dev，那么 64 位的版本将被 32 位的版本取代。如果您想要回到 64 位，您需要首先删除 32 位的版本，再重新安装 64 位的版本。当建立拥有 CRYPTO=builtin 选项的 AllJoyn 时，不需要 libssl-dev。

Python v2.6/2.7

注意: 不兼容 Python 3.0，将导致错误。

  1. Python 是大部分 Linux 版本的一个常规部分。您可以通过使用终端窗口运行以下指令了解 Python 是否已经被安装： sh $ which python

如果返回了一个路径（如 /usr/bin/python），说明 Python 已经被安装了。

  1. 否则，打开一个终端窗口并且执行以下命令： sh $ sudo apt-get install python

  2. 如果此安装方法未能给您提供一个正确版本的 Python， 安装 required version.

SCons

SCons 是一个建立 AllJoyn 架构的软件构建工具。SCons 在大部分 Linux 版本中是一个默认包。

打开一个终端窗口并执行以下命令：

$ sudo apt-get install scons

SCons v2.3 验证 AllJoyn 的构建。

OpenSSL（可选）

OpenSSL 是提供安全网络通信和加密方式的一种开源工具包。AllJoyn 只使用 OpenSSL 的加密方式。 建议您使用最新版本的 OpenSSL.

打开一个终端窗口并且执行以下命令：

$ sudo apt-get install libssl-dev

git

Git 是一个连接源代码文件夹的工具。AllJoyn 源代码存储在一套 git projects中。

打开一个终端窗口并执行以下命令：

$ sudo apt-get install git-core

Repo

Repo 是用于管理多个 git 工程的工具。AllJoyn 源代码以一系列 git 工程的形式存储，可以通过使用 Google's repo tool 单独或批量克隆。此工具不是必需的，但强烈建议使用。

  1. 打开一个终端窗口并且执行以下命令安装 curl：

sh $ sudo apt-get install curl

  1. 导航至根目录并且执行以下命令下载 repo。

sh $ curl https://storage.googleapis.com/git-repo-downloads/repo > ~/bin/repo

  1. 将 repo 拷贝至 /usr/local/bin 并执行以下命令使其可执行：

sh $ sudo cp repo /usr/local/bin $ sudo chmod a+x /usr/local/bin/repo

Uncrustify

Uncrustify 是一个用于保持 AllJoyn 代码库编程风格一致的代码格式工具。这不是构建 AllJoyn 所必需的，但如果您希望对 AllJoyn 工程的代码进行改动，您应该配置并使用这个工具。

注意: Uncrustify v0.61 支持 AllJoyn v15.05 和更高版本。早期版本的 AllJoyn 需要 uncrustify v0.57。由于使用了特定版本的 uncrustify 规定了现有 AllJoyn 代码的格式，当不使用 WF＝OFF 选项构建程序时，使用其它版本的 uncrustify 可能会遇到不可预料的构建错误。

有两种方式安装 Uncrustify。

  • 下载源并构建和安装 Uncrustify：

sh $ mkdir $HOME/uncrustify # for example $ cd $HOME/uncrustify $ git clone http://github.com/bengardner/uncrustify.git $ # or use $ #git clone git://uncrustify.git.sourceforge.net/gitroot/uncrustify/uncrustify $ cd uncrustify $ git checkout uncrustify-0.61 $ # or for v0.57: $ #git checkout uncrustify-0.57 $ ./configure $ sudo make install

注意: 在某些情况下，Uncrustify 在最新的 Ubuntu 版本中构建不成功。试着进行以下改变以构建 Uncrustify。

```sh diff --git a/src/uncrustify.cpp b/src/uncrustify.cpp index 2635189..7aba76d 100644 --- a/src/uncrustify.cpp +++ b/src/uncrustify.cpp @@ -32,6 +32,7 @@ #ifdef HAVE_STRINGS_H #include / strcasecmp() / #endif +#include

/ Global data / struct cp_data cpd; ```

  • 在 Ubuntu 上安装 Uncrustify v0.57 包。
  • 前往 http://packages.ubuntu.com/precise/uncrustify。
  • 在网站中点击 "Download uncrustify" 列表选择您设备的架构。
  • 在选择后打开的网页中，选择一个基于您位置的镜像，下载 .deb 包。
  • 根据情况，使用以下任一命令安装此包。

sh $ sudo dpkg -i uncrustify_0.57-1_amdd64.deb $ sudo dpkg -i uncrustify_0.57-1_i386.deb 目前暂不支持 Uncrustify v0.61。

Doxygen

Doxygen tool从源代码生成 HTML 文档。它不用于建立 AllJoyn 代码。

打开一个终端窗口并执行以下命令：

$ sudo apt-get install doxygen

Graphviz

Graphviz Dot tool把类的层级结构通过图表展示。Doxygen 会使用它。

打开一个终端窗口并执行以下命令：

$ sudo apt-get install graphviz

TeX Live

TeX Live 提供 LaTeX 二进制文件和针对 Linux 的样式表。这个可选工具可用于从源中生成 PDF 格式的 AllJoyn API 的文档。当然不生成文档，也可以编译 AllJoyn 架构。 如果您希望生成 PDF 文档，请安装 TeX Live。

$ sudo apt-get install texlive

Gecko SDK

Gecko SDK（又名 XULRunner SDK）仅在构建 AllJoyn JavaScript 插件时是必须的。对于其他情况，这个部分是可选的。

插件是针对 1.9.2 版本开发的，尽管它也可能适用于更早的版本。在 64 位的 Linux 中，也可以下载 32 位的版本。（因为只是用了 SDK 的标头）

Install Java 安装 Java

使用 Java 6 或更高版本构建基于 Linux 平台的 AllJoyn 架构。

重要: 使用 apt-get 安装 java 将下载 open-jdk 而不是 sun-jdk。AllJoyn 架构需要使用 sun-jdk。

使用任一以下两种机制，安装 Java。

在 Ubuntu 12.04 或更早版本上安装 Java 6

  1. 安装 Java 6

sh $ sudo add-apt-repository "deb http://archive.ubuntu.com/ubuntu lucid partner" $ sudo apt-get update $ sudo apt-get install sun-java6-jdk

  1. 安装 junit 3.8 或更新版本（需要使用 junit 构建 AllJoyn Java 绑定）
  2. 导航至 (https://github.com/junit-team/junit/wiki/Download-and-Install)。
  3. 下载 jar 文件 "junit-4.9.jar"。

  4. 从下载文件夹将其拷贝至 usr/share/java/junit-4.9。

    sh $ sudo cp junit-4.9.jar /usr/share/java/

  5. 如果您想要运行 junit 测试，安装 Apache Ant 构建工具（仅用于运行 junit tests，不能由于构建 AllJoyn 架构）

sh $ sudo apt-get install ant

在 Ubuntu 12.04 或更新版本上安装 Java 6

在 Ubuntu 12.04 Precise Pangolin 版本中，不再支持 partner 文件夹。您必须根据以下指导手动安装 Java。

  1. 从 Java SE 6u32 Downloads 中，根据相应 cpu 类型，下载 JDK bin 文件。

  2. 使用 chmod 让文件可执行：

sh $ chmod +x jdk-6u32-linux-x64.bin

  1. 解压缩 bin 文件：

sh $ ./jdk-6u32-linux-x64.bin

  1. 把解压缩的文件夹移动到 /usr/lib/jvm/：

sh $ sudo mv jdk1.6.0_32 /usr/lib/jvm/

  1. 把新安装的 Java 加入选择列表：

sh $ sudo update-alternatives --install /usr/bin/javac javac /usr/lib/jvm/jdk1.6.0_32/bin/javac 2 $ sudo update-alternatives --install /usr/bin/java java /usr/lib/jvm/jdk1.6.0_32/bin/java 2 $ sudo update-alternatives --install /usr/bin/javaws javaws /usr/lib/jvm/jdk1.6.0_32/bin/javaws 2

  1. 选择默认 Java：

sh $ sudo update-alternatives --config javac $ sudo update-alternatives --config java $ sudo update-alternatives --config javaws

  1. 检查 Java 版本以确认它被正确安装：

sh $ java -version

会返回类似这样语句：

sh java version "1.6.0_26" Java(TM) SE Runtime Environment (build 1.6.0_26-b03) Java HotSpot(TM) 64-Bit Server VM (build 20.1-b02, mixed mode)

  1. 验证所有指向新 Java 位置的 symlinks。

sh $ ls -la /etc/alternatives/java*

9.（可选，但建议）在 Mozilla Firefox （或是 Chrome）中，开启插件功能。

  • 为 64 为 jdk：

    sh $ sudo update-alternatives --install \ /usr/lib/mozilla/plugins/libjavaplugin.so mozilla-javaplugin.so \ /usr/lib/jvm/jdk1.6.0_32/jre/lib/amd64/libnpjp2.so 2 $ sudo update-alternatives --config mozilla-javaplugin.so

  • 为 32 为 jdk：

    sh $ sudo update-alternatives --install \ /usr/lib/mozilla/plugins/libjavaplugin.so mozilla-javaplugin.so \ /usr/lib/jvm/jdk1.6.0_32/jre/lib/i386/libnpjp2.so 2 $ sudo update-alternatives --config mozilla-javaplugin.so

通过访问 http://www.java.com/en/download/testjava.jsp 测试 Java web 插件。

  1. 安装 junit 3.8 或更高版本（需要 junit 建立 AllJoyn Java 绑定）
  2. 导航至 https://github.com/junit-team/junit/wiki/Download-and-Install。
  3. 下载 jar 文件 "junit-4.9.jar" 并将其从 Downloads 文件夹拷贝至 usr/share/java/junit-4.9。

sh $ sudo cp junit-4.9.jar /usr/share/java/

  1. 如果您想要运行 junit 测试，安装 Apache Ant 构建工具（仅用于运行 junit tests，不能由于构建 Alljoyn架构）。

sh $ sudo apt-get install ant

googletest

Google Test 是 Google 用于编写 C++ 测试的架构。Google Test 是 AllJoyn 架构用于测试其 API 的一种 xUnit 测试结构。Google Test 是可选的，但在建立 C++ 单元测试时是必需的。

  1. 打开浏览器并导航至 http://code.google.com/p/googletest/downloads/list。
  2. 从 googletest 下载页面，下载 gtest-1.7.0.zip。
  3. 解压缩 gtest-1.7.0.zip 的内容到已知位置（如 $HOME/gtest/gtest-1.7.0）

重要: 不要使用 apt-install 安装 libgtest-dev。从 code.google.com 下载源代码。

获取 AllJoyn 源

$ cd $HOME
$ export AJ_ROOT = `pwd`/alljoyn # for example
$ git clone https://git.allseenalliance.org/gerrit/core/alljoyn.git

$AJ_ROOT/core/alljoyn

建立 AllJoyn 架构

使用以下命令建立针对 Linux 的 AllJoyn 架构。

$ export JAVA_HOME="/usr/lib/jvm/java-6-sun" # or java-1.5.0-sun
$ export CLASSPATH="/usr/share/java/junit4.9.jar" # for building Java binding
$ export GECKO_BASE=~/xulrunner-sdk # for building Javascript binding
$ cd $AJ ROOT/core/alljoyn

For 32-bit:对于 32 位：

$ scons BINDINGS=<comma separated list(cpp,java,c,js)>

   ex) $ scons BINDINGS="cpp,java"

For 64-bit:对于 64 位：

$ scons CPU=x86_64 BINDINGS=<comma separated list (cpp,java,c,js)>

   ex) $ scons CPU=x86_64 BINDINGS="cpp,java"

注意: 需要使用 SCons 命令的完全列表以建立 AllJoyn 架构，输入 scons -h。

注意: 使用 SCons 变量 BINDINGS 列出您想要建立的语言绑定。举例说明，为了限制仅使用 C++ 构建,使用 BINGDINGS=cpp.在绑定多个语言的情况下，使用逗号分隔。举例说明，使用 Java 和 C++，那么 BINGDINGS=java,cpp。

注意:

  • CLASSPATH 环境变量指定的路径是之前下载的 junit jar 文件的完整路径名。
  • JAVA HOME 环境变量指定的路径是 jdk 文件夹的路径。
  • 为了在 Linux 和 Windows 上建立 Javascript，我们在建立 AllJoyn 架构时需要 GECKO_BASE。
  • 如果您使用早于 AllJoyn 架构 2.6 版本来建立，那么您需要 CPU=x86-64 建立一个 64 位版本的 AllJoyn 架构。

可能的构建错误

" ImportError: No module named argparse" reported when reading "File "../build_core/tools/bin/whitespace.py", line 18". Python does not have the argparse module installed (versions of python 2.7.1 or newer have it installed by default).

$ sudo apt-get install python-setuptools
$ sudo easy_install argparse

建立 API 文档

默认情况下，AllJoyn API 文档不会在构建过程中建立（除 Java 文档之外）。使用以下命令建立 API 文档：

$ scons DOCS=html
$ scons DOCS=pdf

文档将会放置在 /alljoyn_core/docs/html 或 /alljoyn_core/docs/latex.

  • 查阅文档，请通过浏览器打开 /alljoyn_core/docs/html/index.html
  • 查阅 PDF 文档，请通过 PDF 浏览器打开 /alljoyn_core/docs/refman.pdf。

空白符规则检查器

默认情况下，空白符规则检查器不运行。如果您正在对 AllJoyn 的更改，您应该开启空白符规则检查器运行您修改的程序。

$ scons WS=check

如果空白符规则检查器报告了一项违规，它会列出是哪个文件违规了。为了找到违反了 AllJoyn 空白符规则的段落，运行：

$ scons WS=detail

Uncrustify 能够根据空白符规则自动修复您的文件。

$ scons WS=fix

Build variant 建立变量

默认情况下，AllJoyn 架构建立 debug 变量。建立发行版本的 AllJoyn 架构，使用：

$ scons VARIANT=release

Bindings option Bindings 选项

默认 SCons 脚本采用默认方式建立所有的语言绑定。如果您只对某一种语言绑定有意向，BINDINGS 选项可供您选择您希望的语言。

BINDINGS 选项可以用使用逗号列出您所希望用于构建的多种语言。目前，支持的语言有 cpp,c,java 和 js。语言必须采用小写形式，并且语言之间不能有空格。如果依赖项没有被列出，它们也会自动被建立。举例说明，java 需要以 cpp 的建立为基础。如果使用了空白字段，仅会建立核心文件。

举例：

$ scons BINDINGS=java #this will build core files and Java language bindings
$ scons BINDINGS=c,java #this will build C language bindings and Java language bindings
$ scons BINDINGS= #only build the core files alljoyn_core and common

Crypto 选项：

AllJoyn v15.04 在 scons 命令行中加入了 CRYPTO 选项。使用 CRYPTO=builtin，能够除去 OpenSSL 的依赖，进行 AllJoyn 的构建。

$ scons CRYPTO=builtin

使用 OpenSSl 提供的加密方式：

$ scons CRYPTO=openssl

PolicyDB 选项

AllJoyn v14.06 提供了一项新功能。它可以作为一个转发信息的防火墙／筛选器 被编译进 AllJoyn 路由。POLICYDB 选项决定了此项功能是否被加入。该选项可以被设置为 on 或者 off。

AllJoyn 路由的默认规则不包含 PolicyDB。默认情况下，不包含 PolicyDB。

示例：

$ scons POLICYDB=on

建立 C++ 单元测试

AllJoyn 架构目前自带了一套使用 Google Test C++ 架构的单元测试。为了建立单元测试，必须如 googletest 中提到指定 Google Test 源代码的位置。使用 GTEST_DIR 选项指定 Google Test 源代码的位置。

示例：

$ scons GTEST_DIR=$HOME/gtest/gtest-1.7.0

运行 AllJoyn 应用程序

注意: 对于 v2.6 和更高版本，仅有 Bundled Router 模式。

为了保证 Linux 开发平台被正确地建立，使用本章的指导运行 AllJoyn 路由。

随着 AllJoyn v2.6 的发布，不再能够运行分离的独立路由（alljoyn-daemon）。所有的路由程序现在可以被加入任何一个独立的应用程序，这意味着：

  • 您的用户不再需要安装和运行后台服务（守护进程）来运行使用 AllJoyn 架构的应用程序。

  • 每一个您运行的应用程序将有一个它们自己的内置路由。

  • 在命令行中，输入以下命令以运行 AllJoyn 应用程序：

sh $ cd <workspace>/build/{OS}/{CPU}/{VARIANT}/dist/cpp/bin {OS} = linux {CPU} = x86 or x86-64 {VARIANT} = debug or release $ ./bbservice -n com.test

  1. 打开另一个标签并输入以下命令以运行另一个应用程序。

sh $ cd <workspace>/build/{OS}/{CPU}/{VARIANT}/dist/cpp/bin $ ./bbclient -n com.test -d

  1. 在 bbclient 上检查以下输出：

sh Sending "Ping String 1" to org.alljoyn.alljoyn_test.my_ping synchronously org.alljoyn.alljoyn_test.my_ping ( path=/org/alljoyn/alljoyn_test ) returned "Ping String 1"

AllJoyn 路由命令行可执行

注意: 仅对 2.6 之前的版本生效。

v2.6 中引入了路由与应用程序绑定的概念。在此版本之前，在运行 AllJoyn 应用程序时，您需要首先运行 alljoyn-daemon。

作为生成过程的一部分，建立了 alljoyn-daemon 的可执行文件。

  1. 在命令行中，输入以下命令，以分离进程运行 AllJoyn router。

sh $ cd <workspace>/build/{OS}/{CPU}/{VARIANT}/dist/cpp/bin $ ./alljoyn-daemon --internal 变量的可选项如下：

{OS} = linux {CPU} = x86 or x86-64 {VARIANT} = debug or release

此方法采用默认配置开启 AllJoyn 路由。对于大部分用户，以上的命令足够运行 AllJoyn 架构。

  1. 任意时刻按下 Ctrl-c 停止 alljoyn-daemon 进程。
  2. 输入以下命令显示其他选项：

```sh $ ./alljoyn-daemon -h

  alljoyn-daemon [--config-file=FILE] [--print-address]
  [--verbosity=LEVEL] [--no-bt] [--version]

     --config-file=FILE
        Use the specified configuration file.

     --print-address
        Print the socket address to STDOUT.

     --no-bt
        Disable the Bluetooth transport (override config file setting).

     --verbosity=LEVEL
        Set the logging level to LEVEL.

     --version
        Print the version and copyright string, and exit.

```

从以下位置获得不同配置文件的示例：

<workspace>/alljoyn_core/daemon/test/conf.

注意: 不是所有在 daemon/test/conf 文件夹中的配置文件适用于运行 Linux 的电脑。

验证路由正在工作

导航至工程示例目录并用以下命令运行服务和客户端：

$ cd <workspace>/build/{OS}/{CPU}/{VARIANT}/dist/cpp/bin/samples
$ ./basic_service & #this will be a background process; it could be run on its own command-line
$ ./basic_client

运行单元测试

注意: 以下指导只适用于 v2.6 和更高版本的 AllJoyn 架构。

运行 C++ 单元测试

如果在建立代码时制定了 GTEST_DIR 选项，C++ 测试单元会在以下位置被自动建立。

<workspace>/build/{OS}/{CPU}/{VARIANT}/test/cpp/bin.

将会有两个可执行文件：ajtest 和 cmtest。

cmtest

cmtest 测试常用工程的代码，不需要 AllJoyn 路由的运行。cmtest 运行如下：

<workspace>/build/{OS}/{CPU}/{VARIANT}/test/cpp/bin/cmtest

ajtest

ajtest 测试 alljoyn_core 中的代码。为了测试的成功运行，必须运行 AllJoyn 路由器。目前，ajtest 是有局限性的。它不能测试总线到总线（如设备到设备）的连接。ajtest 运行如下：

  1. 开启 alljoyn-daemon(可选查看注释)

sh <workspace>/build/{OS}/{CPU}/{VARIANT}/dist/cpp/bin/alljoyn-daemon --internal

  1. 运行 ajtest。

sh <workspace>/build/{OS}/{CPU}/{VARIANT}/test/cpp/bin/ajtest

关于路径，使用运行 SCons 时特定的 OS，CPU 和 VARIANT 替换 {OS}、{CPU} 和 {VARIANT}。

注意: 如果采用绑定路由的方式构建代码（如，SCons flag BR=on），ajtest 不需要启动分离的 alljoyn-daemon。

运行 Java junit 测试。

junit 测试总是和 Java 绑定同时存在。junit 测试专为测试 Java 绑定而设计。

  1. 从顶部建立文件夹，使用 ant 开始测试。

sh ant test -DOS={OS} -DCPU={CPU} -DVARIANT={VARIANT}

  1. 从以下位置的结果中获得 HTML 版本。

sh <workspace>/build/{OS}/{CPU}/{VARIANT}/test/java/reports/junit/

关于路径，使用运行 SCons 时特定的 OS，CPU 和 VARIANT 替换 {OS}、{CPU} 和 {VARIANT}。

Miscellaneous

Library liballjoyn.so not found

如果返回以下错误：

error while loading shared libraries: liballjoyn.so:
cannot open shared object file: No such file or directory

SCons 脚本建立一个共享库并且与之进行连接。把该库加入 link path。

$ export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:<workspace>/build/{OS}/{CPU}/
   {VARIANT}/dist/cpp/lib

在加入了 library LD_LIBRARY_PATH 后，重新运行报错的程序。
