Run the Notification Sample App - iOS

Prerequisites

Follow steps in the Building - iOS OS X section to build and install the Notification sample app on an iOS device.

You can use the Notification service sample app to act as a Notification Producer, a Notification Consumer, or both a Producer and Consumer, simultaneously. If you run the app as both a producer and consumer simultaneously, you will be able to send and receive notifications on a single iOS device.

Alternately, you can run the app as a Producer on one device, and as a Consumer on another device.

NOTE: Both devices must be on the same Wi-Fi network.

Run the Notification Sample App as a Producer

  1. Launch the Notification sample app on your iOS device.

  1. In the Application Name field, enter a name to be used as the application name by the Notification producer, for example, TestApp.
  2. Click Producer to run in Producer mode.

  1. In the first text field, enter the message you want to send in a notification.
  2. Optionally, enter the message in another language in the second text field. You can use the selector to the right of the text field to choose the second language.
  3. Use the standard TTL, or enter a new one if desired.
  4. Optionally, toggle on the Audio and Icon URL fields.
  5. Next to the Message type label, use the selector to choose a message type of INFO, WARNING, or EMERGENCY.
  6. Press the Send button to send the notification.

NOTE: To receive and view the notification you sent, either run the Notification service sample app as a Consumer on the same device or on a different device.

Run the Notification Sample App as a Consumer

  1. Launch the Notification sample app on your iOS device.

  1. Do one of the following in the Application Name field:
  2. Enter a name to be used as the application name. In this case, the application filters out any notifications that it receives where the app name in the notification does not match the app name in the Application Name field. This feature is to facilitate testing sending and receiving notifications between multiple different applications simultaneously on the same network.
  3. Leave the field blank. In this case, the application will display all notifications that it receives, regardless of which application produced them.
  4. Click Consumer to run in Consumer mode.

The application will now receive notifications.

NOTE: To receive notifications in the app, either run the Notification service sample app as a Producer on the same device or on a different device, as detailed in Run the Notification sample app as a Producer.
